新婚生活スマートホーム ① 自動化の対象 ＆ ② プラットフォーム選定

作成日：2026-06-16 / 対象：畑family 新居（2人暮らし・IT/AIエンジニア ✕ 芸術家・SwitchBot 利用中） 目的：①「何を自動化するのが定番か」を棚卸し（SwitchBot等で電気・エアコン・カーテン…）＋②核となるプラットフォーム（Alexa / Google Home / Apple Home / Matter / SwitchBot）の判断軸を設計して比較。 方針：2026年6月時点の公式・一次情報＋国内レビューを横断。価格・対応状況は変動するため購入前に各公式で再確認。 お二人の優先度（プロフィールから逆算）＝ プライバシー/ローカル動作 ・ 作り込みの自由度 ・ 佇まい（デザイン） ・ 既存SwitchBot資産との相性 ・ ロックイン回避（=Matter重視）。

結論（先に要点）

① 自動化の対象 ―― まずこの「6定番」、次に「応用」、こだわりで「制作環境」

定番6つ＝ 照明 / エアコン / カーテン / 玄関の鍵 / テレビ等IR家電 / スマートプラグ。 ここに 温湿度・人感・開閉センサーを足して「人や時間で自動で動く」状態にするのが王道。お二人ならさらに アトリエの環境ログ（温湿度・CO2）と作品照明（調光調色）、制作/会議モードのシーンまで作ると“らしさ”が出ます（詳細は §1）。

② プラットフォーム ―― 「1つ選ぶ」より「4レイヤーで組む」のが2026年の正解

レイヤー 役割 お二人への推奨
🧠 母艦（ハブ/操作の顔） 音声・アプリ・オートメーションの中枢 Apple Home（プライバシー最良・佇まい・ローカル動作）※AndroidならGoogle Home
🔗 グルー（共通規格） メーカー跨ぎの相互接続 Matter / Thread（ロックイン回避の保険。必ず効かせる）
✋ 手足（デバイス群） 実際に動くモノ SwitchBot（既存資産＋日本の住宅に強い。Matterブリッジ兼用）
🛠️ 拡張（上級者の遊び場） 無制限の自動化・可視化 Home Assistant（エンジニアの本領。任意・後付けでOK）

👉 本命構成：Apple Home を母艦に、Matter で繋ぎ、SwitchBot を手足にする。 物足りなくなったら Home Assistant を一台足して青天井に。「今いちばん賢い会話AIで操作したい」なら、母艦はそのままに Echo（Alexa+）か Google スピーカー（Gemini for Home）を“声の窓口”として1台だけ併設するのが、ロックインせず良いとこ取りできる形です。

⚠️ 分岐の鍵は スマホがiPhoneかAndroidか の1点。iPhone中心＝Apple Home、Android中心＝Google Home に母艦を置き替えるだけで、下の構成思想はそのまま使えます。

第1部：何を自動化するのが「定番」か

1.2 「何を起点に動かすか」＝トリガーの型（ここが自動化の本体）

デバイスよりトリガー設計が肝。定番は5系統：

  1. 時間：朝7時にカーテン開＋エアコン、24時に全消灯＋施錠。
  2. 人の在/不在（人感・位置情報）：帰宅で点灯・解錠・空調ON／全員外出で一括OFF＋掃除起動。
  3. 環境（温湿度・照度・CO2）：室温28℃で冷房、湿度70%で除湿、暗くなったら点灯、CO2高で換気喚起。
  4. 状態変化（開閉・施錠）：玄関ドア開→廊下点灯、窓開→その部屋の空調停止。
  5. 音声・手動シーン：「いってきます」「おやすみ」「制作モード」のひと声 or ボタン一発。

1.3 お二人向けシーン例（そのまま使える“レシピ”）

  • 🌅 おはよう：7:00にカーテン自動オープン → 寝室を暖色で徐々に点灯 → リビングのエアコンON → スピーカーが天気と予定を読み上げ。
  • 🏠 ただいま（位置情報 or 解錠）：玄関＆廊下点灯 → エアコンを快適温度へ → 好きなプレイリスト。
  • 🎨 制作モード（アトリエ）：高演色で昼白色5000K・調光100% → 余計な通知を抑制 → 換気/CO2監視ON → サーキュレーター弱。
  • 💻 会議モード（在宅ワーク）：顔が明るく映る前面光 → 「会議中」をディスプレイ/ライトで家庭内に表示 → 生活音家電（掃除機等）を一時停止。
  • 🌙 おやすみ：全消灯（寝室だけ暖色1%常夜灯）→ 玄関施錠 → 戸締り未施錠を通知 → 空調を就寝・静音モードへ → カーテン全閉。
  • 🚪 いってきます：全消灯＋施錠＋空調OFF → ロボット掃除機起動 → 防犯（在宅偽装のランダム点灯を夕方に）。

1.4 IT/芸術家“ならでは”の伸びしろ

  • 作品が主役の照明：演色性(Ra/CRI)の高い電球＋スポットで“ギャラリー化”。時間帯で色温度を自然光に追従させると作品の見え方が安定。
  • アトリエの環境管理＝作品保存：温湿度・CO2を記録し、しきい値で通知/換気。紙・絵具・楽器・機材の保管に効く。
  • ダッシュボード/ログの可視化：Home Assistant で全センサーを1画面に。電力・室温・在室の推移をグラフ化＝エンジニアの遊び場かつ実益（光熱費の最適化）。
  • 自然言語で操作：2026年は各社が会話AIを搭載（§2）。「寒いから少し暖めて」レベルの曖昧な指示が通り始めた年。

1.5 落とし穴（先に知っておくと事故らない）

  1. 賃貸/原状回復：SwitchBotのロック・スイッチ・カーテンは概ね両面テープで後付け（穴あけ不要）。退去時に剥がせるか粘着跡の可否を確認。新築持ち家でも、壁スイッチ系は配線を傷めない後付け型が無難。
  2. 赤外線は“見通し・1方向”が弱点：ハブとエアコン/TVの間に遮蔽物があると届かない。家具レイアウト前にハブの定位置を決める。広い家は階・部屋ごとにハブ複数。
  3. 電波の前提：多くが2.4GHz Wi-Fi＋Bluetooth。Matterの真価を出すなら Thread（メッシュ）対応のボーダールーター（HomePod/Apple TV/対応ハブ）があると安定。新居のルーター/メッシュ計画とセットで。
  4. “全自動”の言葉に注意：センサー連動は便利だが、誤作動（人感の死角・ペット・来客）も起きる。まず手動シーン→慣れたら自動化の順が安全。
  5. マイク/カメラの置き場所：寝室・アトリエに常時マイク/カメラを置くかは方針を決める（§2.7 プライバシー）。

第2部：核となるプラットフォームの選定

2.1 結論（本命構成）

母艦=Apple Home / グルー=Matter・Thread / 手足=SwitchBot / 拡張=Home Assistant。 理由は、お二人の優先度（プライバシー・佇まい・自由度・既存資産・ロックイン回避）に最も多く当たるから。Androidが主なら母艦を Google Home に置き替え（Gemini for Home が国内早期アクセス開始済で会話AIが強い）。今いちばん賢い会話AIを声で使いたいなら、母艦はそのままに Echo(Alexa+) か Google スピーカーを“声の窓口”として1台だけ足すのが、ロックインせず良いとこ取りできる構成です。

2.2 まず地図：4レイヤーで考える（“1社に全部寄せない”）

2026年は Matter のおかげで「母艦は1つに決めつつ、デバイスはメーカー横断で選べる」のが当たり前に。だから選択は「どれか1つ」ではなく役割分担で考えるのが正解。

  • 🧠 母艦（エコシステム）：Alexa / Google Home / Apple Home から主に使う1つ。音声・アプリ・オートメーションの“顔”。
  • 🔗 グルー（共通規格）：Matter（接続規格）と Thread（省電力メッシュ通信）。これを効かせておけば、後で母艦を乗り換えてもデバイスは生き残る＝保険。
  • ✋ 手足（デバイス）：SwitchBot が日本の住宅（赤外線家電・後付けカーテン/ロック）に圧倒的に強い。ハブが Matterブリッジを兼ねる。
  • 🛠️ 拡張：Home Assistant（OSS・完全ローカル）。母艦の自動化が物足りなくなった時に“青天井”を足す層。エンジニア向け。

2.3 判断軸の設計（なぜこの軸か／太字＝お二人の最優先）

# 判断軸 なぜ重要か（お二人にとって）
① プライバシー / ローカル動作 AI当事者。音声・在室・間取り・施解錠ログを誰がどこに持つか。クラウド依存度と広告/商流との距離
② オートメーションの自由度 エンジニア。条件分岐・変数・スクリプト・APIまで作り込めるか（“できることの天井”）
③ デザイン / UI・佇まい 芸術家＆新居。アプリの美しさ、ハブ/スピーカーの存在感、表示の上品さ
④ 既存SwitchBot資産との相性 既に運用中。ハブ・ロック・カーテン等がそのまま活きるか
⑤ 日本語＆音声AIの賢さ（2026） 曖昧な自然言語をどれだけ理解するか。国内対応の早さ
⑥ 将来の拡張性 / ロックイン回避 Matter成熟度・乗り換え耐性。長く住む前提での投資保護
⑦ 入口コスト / 学習コスト 始めやすさ、家族（将来含む）が使えるシンプルさ

2.5 各プラットフォーム詳細

🟦 Apple Home（＋homeOS / HomePad）――【本命の母艦・iPhone前提】

  • 強み：プライバシー最良（ローカル/オンデバイス志向）、Homeアプリの統一感・佇まいが芸術家・新居に好相性、オートメーションは Home Hub（HomePod/Apple TV）上でローカル実行＝ネット断でも動く堅さ。Matterコントローラ＆Threadボーダールーターも兼ねる。
  • 2026の動き：WWDC 2026で homeOS と新ハブ 「HomePad」（約7インチ画面・A18・Face IDで“誰が部屋にいるか”認識）をプレビュー、2026年秋（〜9月）想定。次世代Siriの仕上がりが発売の鍵（Siri刷新はこの構成の“弱点である会話力”を一気に底上げする本命イベント）。
  • 弱点：ネイティブ対応デバイスは最少（→Matter＋SwitchBotブリッジで補う前提）。オートメーションには家に常時稼働のHome Hubが必須。現状のSiriは会話力で他社に劣る（2026後半に改善見込み）。
  • 出典：Macworld: 2026 Apple smart home ／ MacObserver: HomePad ／ TechTimes: WWDC2026 homeOS

🟧 Amazon Alexa（＋Alexa+）――【“声とAI”の窓口に最適・サブ推奨】

  • 強み：対応デバイス 約30万で最広、入口が最も安く簡単。Alexa+（LLM搭載）は2026年時点で会話AIの完成度が頭一つ抜けている（曖昧/複数指示に強い）。
  • 2026の動き：Alexa+は $19.99/月（Prime会員は追加無料）。ただし本格版は米国先行で、日本は未提供（無印Alexaは日本語対応）。国内提供は2026年半ば〜2027年初が目安。
  • 弱点：クラウド寄りで広告/購買の商流に近い＝プライバシー軸②③を最優先するお二人には“母艦”より**“声の窓口/サブ”**が妥当。UIは多機能ゆえ雑多になりがち。
  • 出典：note: Alexa+ 全貌(2026) ／ Ai-Bo: Alexa+ 日本導入・料金

🟩 Google Home（＋Gemini for Home）――【Android主なら本命の母艦】

  • 強み：自然言語の理解が良く、Gemini for Home で急伸。検索/予定/地図などGoogle経済圏と滑らか。価格も手頃。
  • 2026の動き：Gemini for Home の早期アクセスが日本でも開始（2026/4/8〜、Google Homeアプリ v4.12+）。16カ国・7言語（日本語含む）。よく使う照明操作など応答を最大40%高速化。一部高度機能（Gemini Live・カメラ履歴検索等）は Google Home Premium（有料）。春に新Nest Cam／新スピーカー。
  • 弱点：プライバシーはクラウド寄り（Google経済圏）。母艦をAppleにする場合は“声の窓口/サブ”として1台併設が無難。
  • 出典：Google Japan: Gemini for Home 早期アクセス ／ SBビット: 国内アクセス開始

🔗 Matter / Thread ――【グルー＝必ず効かせる“保険”】

  • 何者か：Amazon/Google/Apple/Samsung等が策定した共通接続規格(Matter)と省電力メッシュ通信(Thread)。Matter対応なら母艦を問わず基本操作が通る＝ロックイン回避の本命。
  • 2026の現状：Matter認証 700製品超、Thread認証 1000製品超へ拡大。Thread 1.4で複数ボーダールーターの相互運用が前進、Matter 1.5でカメラ対応追加。ただし各社の実装は足並みが不揃い（1.2止まりのプラットフォームも）で「全機器が完璧に繋がる」段階ではない＝“繋がる保険”として使いつつ、肝心デバイスは母艦ネイティブ対応も確認が現実解。
  • 出典：matter-smarthome: 2026 Status ／ MakeUseOf: Matter & Thread 2026

✋ SwitchBot ――【手足＝既存資産・日本の住宅に最強・Matterブリッジ兼用】

  • 強み：赤外線家電（エアコン/TV）・後付けカーテン・スマートロックなど“日本の賃貸/既設住宅で効く”製品が網羅的。国内サポートも手厚い。ハブが Matterブリッジを兼ね、SwitchBotデバイスを Apple Home/Google/Alexa/SmartThings へ橋渡しできる。
  • Matterで“外”に出せるもの（公式）：カーテン/ロールスクリーン、ロック全シリーズ、ボット/リモートボタン、温湿度計/CO2、人感/開閉/水漏れセンサー、照明（シーリング/電球/テープ）、プラグミニ、学習リモコン、ロボット掃除機。ブリッジに登録した赤外線家電（エアコン等）もMatter経由で公開可。
    • ⚠️ ただしMatter経由の操作は粒度が粗くなりがち（エアコンの全モード・IRの全ボタン・複雑シーンは結局SwitchBotアプリ側が前提）。“細かい作り込みはSwitchBotアプリ、横断のON/OFFはMatter”の二段構えで考える。
  • ハブの選び方：ハブ3（2025/5/16発売・実勢 約1万円前後※要確認）が現行の中核。2.4型カラー液晶＋物理ダイヤル「Dial Master」、温湿度・照度に加え人感センサー内蔵、赤外線学習、Bluetoothブリッジ、Matter対応。スマホを開かず本体だけで家電/シーンを切替できるのが新点。ハブ2／ハブミニ(Matter対応) も引き続きMatterブリッジ可。
  • 出典：SwitchBot: Matter対応 ／ SwitchBot ハブ3 ／ すまほん: ハブ3レビュー

🛠️ Home Assistant ――【拡張＝エンジニアの“青天井”・任意】

  • 強み：OSSで完全ローカル可＝プライバシー最強、連携2000以上、条件分岐・変数・スクリプト・APIまで作り込み自由（②軸の天井がない）。HomeKit Bridge 経由で自作自動化を Apple Home にも露出できる＝母艦と共存。
  • 弱点：Raspberry Pi/Mini PC等の機材と学習コスト。家族全員が触る“顔”には不向き（顔はApple Home、裏方をHome Assistant、が綺麗）。
  • 位置づけ：最初は不要。SwitchBot＋Apple Homeで運用 → 「もっと凝りたい」となったら後から1台足すのが最適。IT/AIエンジニアの趣味と実益（光熱費可視化・独自AI連携）が両立。

2.6 お二人のケースへの当てはめ／最終提案

  1. プライバシー最優先（AI当事者） → 母艦は Apple Home（ローカル/オンデバイス）。突き詰めたい裏方は Home Assistant（完全ローカル）。声の窓口を足すなら設定でデータ共有を絞る。
  2. 作り込みたい（エンジニア） → まずは母艦のオートメーション、足りなくなったら Home Assistant で青天井。
  3. 佇まい（芸術家・新居） → Homeアプリ＆Apple端末の統一感、SwitchBotの白基調がインテリアに馴染む。
  4. 既存SwitchBot＋日本の家 → 手足は SwitchBot 継続。ハブを Matterブリッジとして効かせる。
  5. ロックイン回避 → Matter/Thread を必ず有効化。母艦を将来乗り換えてもデバイスは生存。

最終提案（本命）

  • 🥇 母艦：Apple Home（iPhone中心の前提）。Home Hubに HomePod系 or Apple TV。秋の HomePad/homeOS は“買い増し候補”として注視。
  • 🔗 グルー：Matter / Thread を必ず有効化（保険）。
  • ✋ 手足：SwitchBot（ハブ3を中核）＋必要に応じ高演色照明・カーテン・ロック。
  • 🛠️ 拡張：Home Assistant は「物足りなくなったら」後付け（最初は無くてよい）。
  • 🗣️ 声/AIの窓口（任意）：会話AIを今すぐ堪能したいなら Echo(Alexa+※国内提供待ち) か Google スピーカー(Gemini for Home/国内開始済) を1台。母艦は替えずサブとして。

分岐（Android中心の場合）：🥇母艦を Google Home に置換（Gemini for Homeが国内で強い）。残り（Matter/SwitchBot/Home Assistant）の思想はそのまま流用可。

2.7 落とし穴（買う前に必ず）

  1. Apple Homeはオートメーションに“常時稼働のHome Hub”が必須（HomePod/Apple TV）。これが無いと自動化・外出先操作・Matter制御ができない。母艦をAppleにするならハブ端末を1つ確保。
  2. Matter経由＝基本操作のみになりがち。エアコンの全モード・IR全ボタン・複雑シーンは結局各社純正アプリ（SwitchBotアプリ等）が要る。「横断はMatter、作り込みは純正」の二段構え前提。
  3. 会話AIの“本番”はこれから：Alexa+は国内未提供、AppleのSiri刷新は2026後半。今すぐ最強の会話AIが要るなら**Gemini for Home（国内開始済）**が一歩先。ただし母艦選定を会話AIだけで決めない（数ヶ月で勢力図が動く領域）。
  4. プライバシーの実運用：寝室/アトリエに常時マイク・カメラを置くか方針決め。Alexa/Googleはクラウド寄り＝データ共有設定の見直しを。最重視なら**Apple Home＋Home Assistant（ローカル）**に寄せる。
  5. 電波・設置：赤外線は見通し1方向、Thread/Matterはボーダールーター（HomePod/Apple TV/対応ハブ）があると安定。新居のWi-Fi/メッシュ計画とハブ定位置を間取り段階で決める。
  6. 賃貸/原状回復：ロック・スイッチ・カーテンは後付け（粘着）可否と退去時の剥がしやすさを事前確認。
