## 5. 手順（今夜）

### Step 1. ミラーリング準備
1. **iPhoneをロック**（画面オフ）。
2. Mac で **iPhoneミラーリング**を起動／再接続（「iPhone in Use」窓なら **Connect**）。
3. ミラーリング窓を**第2ディスプレイに単独・最前面**（背後に何も重ねない）。
4. iPhoneは机に置いて**触らない**。**集中モードON**推奨。

### Step 2. Pairyを「最新の先頭」へ
1. ミラーリング窓内で Pairy → **Question card** を開く。
2. **一番上(最新)** まで戻す。
   - 💡 画面最上部の**ステータスバーをクリック**すると一気に最上部へジャンプ。
3. 先頭が「**同性で好きな芸能人はだれ？(2023/10/02)**」付近であることを確認。

### Step 3. 自動キャプチャ実行
ターミナルで（出力は run2 に）:
```bash
OUT=/Users/koki/pairy_qa_rescue/shots/run2 /Users/koki/pairy_qa_rescue/capture_loop.sh
```
- 実行したら**マウス・iPhoneに触れず放置**（数分）。
- **正常終了**: 最古まで到達し「終端に到達（最古まで取得完了）」と表示。
- **切断時**: 「⚠️ ミラーリング切断画面を検知。中断」と表示して停止 → Step 1 から**再接続→最上部に戻して再実行**（`OUT=…/run3` 等にすると安全）。
- 進捗ログ: `tail -f /Users/koki/pairy_qa_rescue/shots/capture.log`
- 💬 **Claudeと一緒にやる場合**: 準備して「ゴー」と言えば、起動・監視・以降のOCR/Sheet化まで進めます。

### Step 4. 撮り切れたか確認
- 最後のショットを開き、**一番古い質問（2015〜2016頃）**まで写っているか確認。
- 全ショットがPairyカード（赤ヘッダー）で、**`iPhone in Use` 画面が混ざっていない**こと。
- 不足区間があれば、その区間だけ**重複を持たせて**追撮。

### Step 5. OCR → 構造化（Claudeが実施）
- 全ショットを Apple Vision OCR → 重複除去 → 各カードを抽出:
  **質問 / Koki回答 / Koki日付 / Hikari回答 / Hikari日付 / 💬コメント数 / 元画像名**
- 参考コマンド（OCRのみ）:
  ```bash
  "$HOME/Library/Mobile Documents/com~apple~CloudDocs/Others/ExportedPairy/qa_rescue/vision_ocr" \
    /Users/koki/pairy_qa_rescue/shots/run2/*.png > /Users/koki/pairy_qa_rescue/ocr_run2.txt
  ```
- ローカルに `qa.csv` 生成 → スクショと突合して目視補正。

### Step 6. Drive格納 & Googleシート作成（Claudeが実施・gog使用）
- 指定フォルダ（`14gZ…`）へ:
  - **スクショ画像を一括アップロード**（サブフォルダにまとめる）
  - **Q&A をネイティブ Google スプレッドシート**として作成・投入（並びは古い順）
- ※そのフォルダに書き込める Google アカウントで実行（gog登録アカウントを確認。**接続承認が要る場合あり**）。

### Step 7. コメント取得（💬≥1 のカードのみ・本編と分離）
- Step 5 のOCRで「💬≥1」のカードを一覧化（run1時点で 💬1, 💬2 を確認済み）。
- コメントは**詳細画面に入らないと見えない**ため、**本編キャプチャ完了後**に分離して取得:
  - 該当カードを開く（手動 or 自動）→ コメント画面を撮影 → OCR → Sheetのコメント列へ追記。
  - 🔑 本編スクロールには**絶対に割り込ませない**（再開位置がズレないように）。

## 6. トラブルシュート

| 症状 | 対処 |
|---|---|
| 「iPhone in Use」で切断 | iPhoneをロック → Connect → **最上部に戻して再実行**。集中モードON |
| 窓が見つからない(NOT_FOUND) | ミラーリングを最前面に。`/Users/koki/pairy_qa_rescue/mirror info` で座標確認 |
| スクロールしない | Macのアクセシビリティ権限（ターミナル/Claude）を確認。実行中カーソルに触れない |
| 真っ黒/別ウインドウ混入 | 窓を**第2ディスプレイで単独・最前面**に。フルスクリーンの別アプリと別Spaceにしない |
| 長文カードが途中で切れる | `STEP=-250`（重なり増）で再実行 |
| 読み込みが遅い | `WAIT=3`（待ち増）で再実行 |

## 7. パラメータ（`capture_loop.sh`）

| 変数 | 既定 | 意味 |
|---|---|---|
| `OUT` | `…/shots/run1` | 保存先（今夜は `…/shots/run2`） |
| `STEP` | `-300` | 1スクロール量（px・負=下／小さいほど重なり多＝安全） |
| `WAIT` | `2` | スクロール後の追加読み込み待ち（秒） |
| `STOP_AFTER` | `5` | 連続でこの枚数だけ画面が同一なら「最古に到達」と判定 |
| `MAX` | `1000` | 暴走防止の上限枚数 |
| `BAD_HASH` | 切断画面md5 | これを検知したら中断（誤完了防止） |

例:
```bash
OUT=/Users/koki/pairy_qa_rescue/shots/run2 STEP=-250 WAIT=3 \
  /Users/koki/pairy_qa_rescue/capture_loop.sh
```
